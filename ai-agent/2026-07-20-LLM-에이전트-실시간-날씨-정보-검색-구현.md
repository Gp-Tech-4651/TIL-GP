### Context
* 사용자가 "오늘 밤 날씨는 어때?"와 같이 실시간 정보가 필요한 질문을 던졌을 때, LLM의 정적 학습 데이터 한계를 극복하기 위해 외부 검색 API(SerpAPI)를 연동하여 실시간 날씨 정보를 획득하는 과정을 분석함.
* 특정 지역이 명시되지 않은 경우 기본 위치를 서울로 가정하고, 획득한 기상 데이터를 가공하여 사용자에게 유용한 답변을 생성하는 에이전트 워크플로우를 설계함.

### Core
* 에이전트가 외부 검색 도구(SerpAPI)를 호출하여 기상 정보를 획득하고, 화씨 온도를 섭씨 온도로 변환하여 제공하는 파이썬(Python) 예시 코드임.

```python
import os
from serpapi import GoogleSearch

def get_seoul_weather():
    # SerpAPI 파라미터 설정 (영어 쿼리로 최적화된 검색 수행)
    params = {
        "engine": "google",
        "q": "Seoul weather",
        "api_key": os.getenv("SERPAPI_API_KEY")
    }
    
    search = GoogleSearch(params)
    results = search.get_dict()
    
    # API 응답 데이터 가공
    # 실제 응답 예시: 기온 82°F, 습도 80%, 강수확률 50%, 날씨 'Cloudy'
    temp_f = 82
    temp_c = round((temp_f - 32) * 5 / 9, 1) # 화씨 -> 섭씨 변환
    humidity = "80%"
    precipitation = "50%"
    condition = "흐림(Cloudy)"
    
    return {
        "temperature_c": temp_c,
        "humidity": humidity,
        "precipitation": precipitation,
        "condition": condition
    }
```

* 최종적으로 LLM 에이전트가 작성한 사용자 응답 예시임.
* 오늘 밤 서울의 날씨는 기온 약 27.8°C로 다소 후덥지근하며, 하늘은 흐린 상태입니다. 습도는 80%로 높고 강수 확률은 50%이므로, 외출 시 가벼운 우산을 준비하시는 것을 권장합니다.

### Insight
* LLM 에이전트가 위치나 시간대가 모호한 자연어 질문을 받았을 때 시스템 프롬프트를 통해 기본 위치를 지정하거나 사용자 콘텍스트를 동적으로 파악하는 프롬프트 엔지니어링이 중요함을 파악함.
* 외부 API에서 제공하는 화씨(Fahrenheit) 등의 단위를 사용자의 문화적 배경에 맞춰 섭씨(Celsius)로 자동 변환하는 후처리 로직을 도구 내부에 내재화함으로써 LLM의 할루시네이션(Hallucination)과 연산 오류를 방지함.
* 단순 기상 수치 제공에 그치지 않고, 강수 확률 50%와 흐린 날씨 정보를 조합해 "우산을 챙기라"는 식의 실용적인 권장 행동을 생성하는 것이 에이전트의 완성도를 높이는 핵심 요인임.
