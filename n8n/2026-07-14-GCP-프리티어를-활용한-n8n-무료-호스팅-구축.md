### Context
n8n을 로컬 환경이 아닌 24시간 가동되는 클라우드 환경에서 운영하고자 할 때, 유료 서버 비용에 대한 부담이 발생합니다. 이를 해결하기 위해 구글 클라우드 플랫폼(GCP)에서 제공하는 Always Free 등급인 e2-micro 인스턴스를 활용하여 비용 없이 인프라를 구축하고, Docker 기반의 컨테이너 환경으로 관리 효율성을 확보하는 방안을 정리합니다. 특히 Ngrok을 활용하면 복잡한 네트워크 설정 없이 외부 웹훅(Webhook) 통신을 안전하게 수행할 수 있습니다.

### Core
GCP e2-micro 인스턴스에 Docker와 Docker Compose를 설치한 후, 아래와 같이 n8n을 구동합니다.

* `docker-compose.yml` 설정 예시:
```yaml
version: '3.8'
services:
  n8n:
    image: docker.n8n.io/n8nio/n8n
    restart: always
    ports:
      - "5678:5678"
    environment:
      - N8N_HOST=localhost
      - N8N_PORT=5678
      - N8N_PROTOCOL=http
      - NODE_ENV=production
    volumes:
      - ~/.n8n:/home/node/.n8n
```

* Ngrok 터널링 실행 명령:
```bash
# 로컬 포트 5678을 외부로 노출
ngrok http 5678
```

### Insight
GCP e2-micro 인스턴스는 RAM 자원이 매우 제한적(약 1GB)이므로, n8n 구동 시 메모리 부족 현상이 발생할 수 있습니다. 이를 방지하기 위해 스왑 메모리(Swap memory) 설정을 권장합니다. 또한, Ngrok은 무료 플랜 사용 시 재시작마다 도메인이 변경되므로, 지속적인 웹훅 서비스 운영을 위해서는 고정 도메인을 제공하는 Ngrok 유료 플랜이나 Cloudflare Tunnel 사용을 검토하는 것이 좋습니다. 기술적으로 Docker Compose를 사용하면 환경 격리가 용이하여 OS 레벨의 의존성 문제를 방지할 수 있는 장점이 있습니다.
